# Изоляция контуров

## Суть
Контуры — независимые области работы. Данные одного контура **не смешиваются** с другим. Это правило защищает от утечки контекста между бизнесами и проектами.

## Терминология

| Термин | Что это | Примеры |
|--------|---------|---------|
| **Контур** | Целостная область работы со своими людьми, встречами и решениями | продукт, клиентское направление, личная область, книга, поездка |
| **Проект** (vault-папка) | Конкретная инициатива внутри контура. Папка в `01_now/projects/` | `2026-click-ru-faq-modification`, `2026-click-ru-cjm` |
| **Головной проект контура** | Папка хранилища — точка входа контура. Содержит общий `log.md`, `context.md`, `materials/`, `meetings/` | `01_now/ops/product/` для продуктового контура |

Карта контуров и их головных проектов — в скилле `meeting-processing/SKILL.md`.

## Правила

1. **Одно сообщение = один контур.** Не упоминать данные одного контура в рассуждениях о другом. Если контур неясен — остановиться и попросить владельца уточнить контур. Нельзя перечислять найденные контуры, проекты или кандидаты.

2. **Никаких неявных связей.** Агент не строит связи между контурами самостоятельно. Данные в `03_knowledge/` принадлежат конкретному контуру и не используются как контекст для другого.

3. **Кросс-контурный референс — только по явному запросу.** При использовании данных из другого контура — **санировать**: заменить упоминания контура-источника на данные целевого контура. Результат должен выглядеть как самостоятельный артефакт целевого контура.

4. **Поиск — с фильтром по контуру.** Поиск по vault без фильтрации по контуру запрещён. До уточнения контура допускается только навигационный поиск верхнего уровня (`AGENTS.md`, `README.md`, правила, индексы). Чтение содержательных файлов конкретных контуров до уточнения запрещено.

5. **Люди не пересекают контуры.** Один человек в нескольких контурах — обособленная сущность в каждом. Упоминание в одном контуре не основание подтягивать данные из другого.

6. **Встречи принадлежат одному контуру.** Если на встрече по одному контуру упоминается другой — это не создаёт связи. Информация фиксируется в рамках текущего контура.

7. **Сеть контактов — доступ через человека.** `contacts-network/` относится к отдельному контуру контактов. В рабочем контуре можно подтянуть историю с конкретным человеком по `person_id`, но не сканировать весь журнал. Принадлежность человека к контуру — через поле `project_or_domain` в служебной разметке карточки.

8. **Определение принадлежности файла в `03_knowledge/`.** Файлы с префиксом контура в имени или в именованной подпапке → принадлежат контуру. Всё остальное (методологии, общие справочники) → общие знания, доступные из любого контура.
