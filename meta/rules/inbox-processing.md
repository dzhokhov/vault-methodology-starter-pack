# Протокол обработки 00_inbox/

## Суть

`00_inbox/` — зона разбора, не мусор. Файлы попадают сюда из разных источников (почта, мессенджеры, ручной импорт, скрипты). Агент, обработавший файл из inbox, **обязан** записать результат обработки в лог. Без записи в лог обработка считается незавершённой.

Это решает главную проблему: владелец не знает, какие файлы в inbox уже обработаны и безопасны для удаления, а какие ещё не тронуты.

---

## Лог обработки

Файл: `00_inbox/PROCESSING_LOG.md`

### Формат записи

```markdown
## 2026-03-31

- **source:** `00_inbox/task-export/` (234 файла)
  **action:** Импортировано в `01_now/projects/example/tasks.md`
  **result:** Все задачи перенесены, исходники безопасны для удаления
  **status:** ✅ processed
  **agent:** Claude (Cowork session)

- **source:** `00_inbox/2026-03-05-project-meeting.md`
  **action:** Извлечены решения → `01_now/projects/example/log.md`, контакты → `03_knowledge/contacts-network/`
  **result:** Полностью разобрано
  **status:** ✅ processed
  **agent:** Claude (meeting-processing skill)
```

### Статусы

| Статус | Значение |
|--------|----------|
| ✅ processed | Полностью обработано, исходник можно удалить |
| 🔄 partial | Частично обработано, нужна доработка |
| 👀 reviewed | Просмотрено, решено оставить как есть (без переноса) |
| ⏳ queued | Запланировано к обработке, ещё не тронуто |

---

## Обязательные действия агента

### При работе с файлом из 00_inbox/

1. **Обработай** — извлеки ценное, перенеси куда нужно (01_now, 02_domains, 03_knowledge)
2. **Запиши в лог** — добавь запись в `00_inbox/PROCESSING_LOG.md`
3. **НЕ удаляй исходник** — удаление только по решению владельца

### При массовой обработке (дампы, импорты)

1. Укажи общее количество файлов и что именно извлечено
2. Если обработана только часть — статус `🔄 partial` с указанием что осталось

### При vault review

1. Прочитай `00_inbox/PROCESSING_LOG.md`
2. Предложи владельцу удалить файлы со статусом ✅ processed
3. Файлы без записи в логе = необработанные, не предлагать к удалению

---

## Что НЕ нужно логировать

- Автоматические дампы, которые перезаписываются (`messenger-dumps`, `web-dumps`) — у них свой TTL в [cleanup-registry.md](../cleanup-registry.md)
- Email-inbox — управляется через SQLite в zimbra_inbox_fetcher.py
- Файлы, которые агент только прочитал, но не обрабатывал (консультация ≠ обработка)
