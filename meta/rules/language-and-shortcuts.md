# Язык ответа и расшифровка сокращений

## Суть

Владелец читает ответы как человек, а не как агент. Буквенно-цифровые идентификаторы (K6a, T5, R4, M8) без расшифровки и англоязычный профессиональный жаргон (shell, pipeline, stage) превращают ответ в шифр. Правило — детализация и ужесточение [Правила 6а из AGENTS.md](../../AGENTS.md).

## Часть 1 — расшифровка индексных сокращений

### Где применяется

Ко всем контурным идентификаторам с индексом:

| Префикс | Что это | Файл-источник |
|---|---|---|
| `K` | Канонические стадии процесса, если они заведены в проекте | профильный файл процесса внутри проекта |
| `R` | Правила предметной области, если проект использует индексированные правила | профильный файл правил внутри проекта |
| `P` | Уровень приоритета находки (P0 — блокирует следующий шаг, P2 — отложено) | Файлы с находками прогонов |
| `T` | Задача (task) в `tasks.md` активного проекта или контура | `<project>/tasks.md` |
| `M` | Веха (milestone) в `plan.md` | `<project>/plan.md` |
| `B` | Карточка блокера в `plan.md §Blockers` ([AGENTS.md Правило 12](../../AGENTS.md)) | `<project>/plan.md` |
| `A` | Канонический документ или результат внутри процессной карты | профильный файл процессной карты |

### Формат расшифровки

`<индекс> (название или краткая суть)` — **при первом упоминании в сообщении**.

Пример правильно:

> Обобщение R4 (правило VK о разделении кампаний по экономике лида) вошло в инвариант 5 общего контракта этапа K6a (концепция кампании, campaign concept).

Пример неправильно (шифр):

> Обобщение R4 вошло в инвариант 5 shell'а K6a.

### Когда расшифровка повторяется

- при первом упоминании в каждом новом сообщении;
- после долгого пропуска в диалоге (новый чат, возврат после сжатия контекста);
- если сокращение получает новый смысл (пересборка вех, переопределение задач).

В пределах одного сообщения после расшифровки можно ссылаться коротко: «дальше в R4…», «остальные пункты T6…».

## Часть 2 — запрет профессионального жаргона без перевода

### Термины, запрещённые в ответах владельцу без русского перевода рядом

| Английский термин | Русский аналог |
|---|---|
| shell | универсальная оболочка / общий контракт стадии |
| pipeline | рабочая цепочка / конвейер этапов |
| stage | стадия / этап |
| handoff | передача между стадиями |
| gate | контрольная точка / условие перехода |
| entry gate / exit gate | условие входа в стадию / условие выхода из стадии |
| registry | реестр |
| manifest | карта возможностей канала / описание канала |
| fallback | запасной путь |
| trigger (в не-бытовом смысле) | условие срабатывания |
| adapter | реализация под канал / канальная реализация |
| scope | объём работы / границы задачи |
| inventory (в смысле реестра состояний) | опись / аудит-матрица |
| audit (в смысле проверки файлов) | проверка / ревизия |
| milestone | веха плана |
| deliverable | результат / выдача |
| artifact (в смысле «файл») | документ / файл (слово «артефакт» допустимо, но лучше конкретнее) |
| contract (в смысле «документ-контракт») | контракт (допустимо, но при первом упоминании объяснить роль) |
| generalization | обобщение |
| upstream / downstream | предыдущий шаг / следующий шаг; сверху / снизу по цепочке |
| source annotation | пометка источника |
| pending | ожидает / отложено до |
| partial / materialized / declared-only | частично собран / материализован / только объявлен |
| coverage | покрытие |
| rehearsal | репетиционный прогон |
| bounded | ограниченный по объёму |
| evergreen | постоянный (в контексте срока жизни документа) |
| seasonal | сезонный (в том же контексте) |

Список не закрытый. При появлении нового термина в ответе — агент проверяет, есть ли русский аналог, и если есть — использует его.

### Имена файлов и путей — исключение

Имена файлов и путей остаются в оригинале, потому что это технические метки: `universal-routing-contract.md`, `campaign-concept-shell.md`, `shell-materialization-inventory.md`. Но **смысл** файла в чате объясняется по-русски:

Правильно:

> В файле `campaign-concept-shell.md` (общий контракт этапа K6a — концепция кампании) зафиксировано шесть неизменных правил.

Неправильно:

> В `campaign-concept-shell.md` зафиксировано 6 invariants для K6a shell.

## Часть 3 — индустриальный минимум (разрешено без перевода)

Только термины, широко распространённые в русскоязычной рекламной индустрии настолько, что их перевод был бы искусственным:

- `CPC` — цена клика
- `CPL` — цена лида
- `CPM` — цена за тысячу показов
- `CPA` — цена действия
- `CTR` — кликабельность
- `ROAS` — возврат на рекламные расходы
- `ROMI` — возврат на маркетинговые инвестиции
- `LTV` — пожизненная ценность клиента
- `UTM` — параметры разметки ссылок
- `URL` — адрес страницы
- `API` — интерфейс взаимодействия систем (универсально распространён)
- `MCP` — протокол подключения внешних систем к агентам (внутренний термин vault'а, владелец его использует)
- `KPI` — ключевой показатель
- `A/B-тест` — сплит-тест

**Не входят в минимум** (переводить обязательно): SDK, IDE, CI/CD, workflow, payload, schema, endpoint, stakeholder, use-case, pain point, value proposition, follow-up, check-in, sync, stand-up, retro и подобные.

## Часть 4 — проверочный шаг перед отправкой ответа

Перед отправкой агент обязан перечитать свой ответ и убрать:

1. Неразвёрнутые индексные сокращения при первом упоминании в сообщении.
2. Англоязычные термины из запрещённого списка без русского перевода рядом.
3. Смесь русского и английского в одном слове (`shell'а`, `gate'ы`, `blueprint'ом`, `handoff'иться`).
4. Кальки, где русский аналог звучит естественно (`генерализация` → «обобщение», `орфанные объекты` → «объекты без родителя»).

Пункт 3 — особая проблема: агент склонен приклеивать русский суффикс к английскому корню (`pipeline'а`, `shell'ом`). Это запрещено жёстче всего — такой текст нечитаем и одновременно нарушает оба правила.

## Часть 5 — работа с файлами базы знаний

Большая часть файлов в базе знаний исторически написана на смеси языков — это факт, не лицензия.

**Обязательно:**
- в чате владельцу правило применяется независимо от того, как написан исходный файл;
- цитаты из файла при необходимости даются в оригинале, но пояснение — по-русски.

**Не обязательно (мягкая норма):**
- при первом касании файла по другой задаче — по возможности перевести жаргон в новых и изменяемых секциях;
- ретроспективный перевод всех файлов базы знаний автоматически **не** запускается;
- если владелец попросит перевести конкретный файл — это отдельная задача со своим циклом.
