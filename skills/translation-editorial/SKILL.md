# Translation and Editorial


> Навигация: [Индекс скиллов](../README.md)
## Overview

Преобразуй технические входные данные в понятный прикладной контент:
- без жаргона,
- с явным практическим эффектом,
- с действиями на 7 дней.

Скилл отвечает за фазу `signal -> message map -> card/post -> editorial QA`.

## Workflow

### 1) Pick Audience

Назначь primary audience:
- `entrepreneur`
- `operator`
- `specialist`

Если аудитория смешанная — выбери одну primary и укажи secondary.

### 2) Build Message Map

Для каждого сигнала/кейса зафиксируй 5 опорных пунктов:
- Что произошло (факт)
- Что это меняет в работе (механика эффекта)
- Для кого это полезно/рискованно
- Что сделать за 7 дней
- Что НЕ делать

### 3) Produce Artifacts

Используй шаблоны из
[card-templates](./references/card-templates.md).

Минимум:
- `Short card` (для дайджеста)
- `Blog block` (развёрнутый абзац)

### 4) Editorial QA

Прогони чеклист из
[editorial-checklist](./references/editorial-checklist.md).

Если есть сложный термин — добавь короткое объяснение на уровне практики.

## Output

Возвращай 2 блока:
1. `Digest Card`
2. `Blog Paragraph`

Для каждого блока укажи:
- primary audience
- ключевой actionable

## Quality Gates

- Главная мысль понятна из первых 2 предложений.
- Нет беспредметной оценки в стиле «революция/прорыв» без механики.
- Есть конкретный шаг на 7 дней.
- Есть блок «что не делать».
- Нет внутренних противоречий с исходным сигналом.

## Anti-Patterns

- Не пересказывать changelog вместо прикладного смысла.
- Не писать "для всех".
- Не выпускать текст без конкретного next step.
- Не раздувать формат: одна карточка = одна центральная мысль.
